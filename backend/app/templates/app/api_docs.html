HRMS Lite API Docs

This page documents the live Django API and lets you execute requests against the running server.

What this covers

  • Employee creation, listing, deletion, and attendance history
  • Attendance marking and filtered record retrieval
  • Ready-to-run request payloads for local development

Sample date for testing: {{ today }}

Endpoints

GET/api/health/

Checks service and database health.

GET/api/v1/employees/

Returns paginated employees with attendance summary.

POST/api/v1/employees/

Creates an employee using employee_id, full_name, email, and department.

DELETE/api/v1/employees/<employee_id>/

Soft deletes an employee.

GET/api/v1/employees/<employee_id>/attendance/

Returns a single employee and attendance history.

GET/api/v1/attendance/?employee_id=EMP001&start_date={{ today }}&end_date={{ today }}

Lists attendance records with optional filters.

POST/api/v1/attendance/

Marks an employee present or absent for a given day.
